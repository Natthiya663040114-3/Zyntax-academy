Web Concepts Q&A

1. Purpose of the index.html file

The index.html file acts as the main document that the browser loads first when visiting a website. It's usually responsible for rendering the initial layout and acts as the default page if no specific file is requested.

2. What URL was used and what protocol?

To preview the site during development, I accessed it via a URL like http://localhost:5173. Since it doesn't begin with https://, the protocol in use is HTTP, meaning it's not encrypted.

3. Status code for index.html and its meaning

The web server responded with a status code 200. This signifies that the file was found and successfully returned to the client without any errors.

4. How to identify local hosting vs a public site?

When the site is accessed through localhost or an internal IP address, it's a local environment. If accessed via a custom domain or public URL, then it's hosted online for global access.

5. Tools used for checking server response

I used the built-in Developer Tools in the browser, particularly the "Network" tab, which allows real-time tracking of request and response data, including headers and status codes.
